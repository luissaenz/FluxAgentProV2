# Fase 6 — Marketplace de Agentes

> **Objetivo:** Extender FAP para permitir que las organizaciones desarrollen, publiquen y comercialicen sistemas agénticos que otras organizaciones puedan consumir como servicios.

---

## Visión del sistema

- **Infraestructura centralizada:** Todo corre sobre la misma instancia FAP del operador.
- **Propiedad intelectual del creador:** El desarrollador mantiene su IP, el operador provee la infraestructura.
- **Marketplace público:** Cualquier org puede publicar agentes; cualquier org puede descubrir y consumir.
- **Consumo como inputs:** Los outputs de agentes publicados pueden usarse como entrada en otros flows del consumidor.

---

## 01 — Modelo de datos

### Arquitectura: Immutable Snapshots

Cada "publicación" es un snapshot inmutable. Las instalaciones apuntan a una versión específica.

**Tabla principal** (puntero a versión actual):

```sql
-- Agent Templates (puntero a versión actual)
CREATE TABLE agent_templates (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_org_id    UUID NOT NULL REFERENCES organizations(id),
    name            TEXT NOT NULL,
    slug            TEXT NOT NULL,  -- NOT UNIQUE - se combina con org_slug
    category        TEXT NOT NULL DEFAULT 'automation',
    
    -- latest_version_id apunta al snapshot actual
    latest_version_id UUID REFERENCES agent_template_versions(id),
    
    is_published    BOOLEAN DEFAULT FALSE,
    install_count   INTEGER DEFAULT 0,
    run_count      INTEGER DEFAULT 0,
    
    created_at     TIMESTAMPTZ DEFAULT now(),
    updated_at     TIMESTAMPTZ DEFAULT now()
);
```

**Tabla de versiones** (snapshots inmutables):

```sql
-- Agent Template Versions (snapshots inmutables)
CREATE TABLE agent_template_versions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    template_id     UUID NOT NULL REFERENCES agent_templates(id),
    version        TEXT NOT NULL,
    
    -- Configuración inmutable de esta versión
    soul_json       JSONB NOT NULL,
    allowed_tools   TEXT[] DEFAULT '{}',
    input_schema   JSONB DEFAULT '{}',
    output_schema  JSONB DEFAULT '{}',
    
    -- sample_input para pruebas (en _metadata.sample_input)
    tech_description TEXT,
    
    -- Schema fingerprint para detectar breaking changes
    input_schema_hash TEXT,  -- SHA256 del input_schema
    
    is_published   BOOLEAN DEFAULT FALSE,
    created_at     TIMESTAMPTZ DEFAULT now(),
    
    UNIQUE(template_id, version)
);

-- Instalar agente en mi organización
CREATE TABLE agent_installations (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id          UUID NOT NULL REFERENCES organizations(id),
    template_id     UUID NOT NULL REFERENCES agent_templates(id),
    version_id      UUID NOT NULL REFERENCES agent_template_versions(id),  -- Apunta a versión específica
    custom_config   JSONB DEFAULT '{}',  -- Encriptado
    custom_tools   TEXT[] DEFAULT '{}',
    status         TEXT DEFAULT 'active',

    -- Update policy
    update_policy   TEXT DEFAULT 'manual',  -- 'auto', 'manual', 'none'
    
    installed_at   TIMESTAMPTZ DEFAULT now()
);

-- Uso por instalación
CREATE TABLE agent_usage (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    installation_id UUID NOT NULL REFERENCES agent_installations(id),
    task_id         UUID REFERENCES tasks(id),
    runs            INTEGER DEFAULT 0,
    period_month    TEXT NOT NULL,  -- '2026-04'
    created_at      TIMESTAMPTZ DEFAULT now()
);

-- Outputs expuestos para consumo externo
CREATE TABLE agent_outputs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    installation_id UUID NOT NULL REFERENCES agent_installations(id),
    task_id         UUID NOT NULL REFERENCES tasks(id),
    output_key      TEXT NOT NULL,  -- clave dentro del output_schema
    output_data     JSONB NOT NULL,
    created_at      TIMESTAMPTZ DEFAULT now()
);
```

## 08 — Seguridad y Aislamiento de Datos

### Principios fundamentales

1. **RLS estricto**: Toda tabla tiene políticas que impiden acceso cruzado entre orgs
2. **Aislamiento del creador**: El creador NUNCA puede ver datos de usuarios que contratan su agente
3. **Credenciales por instalación**: Cada instalación tiene sus propias credenciales encriptadas
4. **Sin acceso del desarrollador**: El campo `custom_config` solo es visible para la org consumidora

### Políticas RLS por tabla

| Tabla | Política |
|-------|----------|
| `agent_templates` | Público lectura (si publicado), owner CUD |
| `agent_installations` | Solo la org consumidora puede leer/editar |
| `agent_outputs` | Solo la org consumidora que ejecutó |
| `agent_usage` | Solo la org consumidora |
| `custom_config` (en installations) | **Solo consumidor** — creator no tiene acceso |

### Restricción del creador

```sql
-- El creador NO puede ver installations de su template
-- Solo puede ver métricas agregadas (install_count, run_count)

-- Función RPC para métricas solo
CREATE OR REPLACABLE FUNCTION get_template_metrics(template_id UUID)
RETURNS JSONB
SECURITY DEFINER
AS $$
BEGIN
    -- Retorna solo counts agregados, sin datos de usuarios
    RETURN jsonb_build_object(
        'install_count', (SELECT install_count FROM agent_templates WHERE id = template_id),
        'run_count', (SELECT run_count FROM agent_templates WHERE id = template_id)
    );
END;
$$;
```

## 09 — Control de Versiones

### Flujo de versiones

1. **Developer publica nueva versión** → `version` se incrementa
2. **Consumer recibe notificación** de nueva versión disponible
3. **Consumer elige**:
   - ✅ Auto-update: Se aplica automáticamente
   - ⏸️ Manual update: Consumer decide cuándo actualizar
   - ❌ No actualizar: Se queda en versión actual

## 16 — Soluciones de Seguridad y Arquitectura

### 1. Encriptación de custom_config (pgcrypto)

```sql
-- Extensión nativa de PostgreSQL
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- custom_config es BYTEA, no JSONB
ALTER TABLE agent_installations
    ALTER COLUMN custom_config TYPE BYTEA;

-- Funciones de encriptación con clave derivada por org
CREATE OR REPLACE FUNCTION encrypt_config(org_id UUID, config JSONB)
RETURNS BYTEA SECURITY DEFINER AS $$
DECLARE key TEXT;
BEGIN
    key := current_setting('app.master_encryption_key', true) || '-' || org_id::text;
    RETURN pgp_sym_encrypt(config::text, key);
END;
$$ LANGUAGE plpgsql;

CREATE OR REPLACE FUNCTION decrypt_config(org_id UUID, encrypted_config BYTEA)
RETURNS JSONB SECURITY DEFINER AS $$
DECLARE key TEXT; decrypted TEXT;
BEGIN
    key := current_setting('app.master_encryption_key', true) || '-' || org_id::text;
    decrypted := pgp_sym_decrypt(encrypted_config, key);
    RETURN decrypted::jsonb;
END;
$$ LANGUAGE plpgsql;
```

### 3. Slug Collision Handling

```python
# generate_unique_slug() con retry en Python
def generate_slug(name: str) -> str:
    import re
    from unidecode import unidecode
    slug = unidecode(name.lower().strip())
    slug = re.sub(r'[^a-z0-9\s-]', '', slug)
    slug = re.sub(r'[\s]+', '-', slug).strip('-')
    return slug

async def generate_unique_slug(db, name: str) -> str:
    base_slug = generate_slug(name)
    slug = base_slug
    counter = 1
    while await db.fetchval("SELECT 1 FROM agent_templates WHERE slug = $1", slug):
        slug = f"{base_slug}-{counter}"
        counter += 1
        if counter > 100:
            raise ValueError(f"Cannot generate unique slug for '{name}'")
    return slug
```

### 4. Enforcement de límites de uso

```python
async def check_usage_limits(installation_id: str, db) -> dict:
    row = await db.fetchrow("""
        SELECT at.pricing_model, at.runs_included,
               COALESCE(SUM(au.runs), 0) as runs_used
        FROM agent_installations ai
        JOIN agent_templates at ON ai.template_id = at.id
        LEFT JOIN agent_usage au ON au.installation_id = ai.id
        WHERE ai.id = $1 AND au.period_month = to_char(NOW(), 'YYYY-MM')
        GROUP BY at.pricing_model, at.runs_included
    """, installation_id)
    
    if row["runs_included"] == -1:
        return {"allowed": True}
    if row["runs_used"] >= row["runs_included"]:
        return {"allowed": False, "reason": f"Limit exceeded"}
    return {"allowed": True}
```

## 18 — Ejecución y Errores

### 1. run_installed_agent con errores y timeouts

```python
AGENT_EXECUTION_TIMEOUT = 300  # 5 minutos

async def run_installed_agent(self, installation_id: str, input_data: dict) -> dict:
    installation = await self._get_installation(installation_id)
    
    # Validar input contra schema
    validate_schema(input_data, installation["input_schema"])
    
    # Validar tools disponibles
    available = await self._get_org_tools(self.org_id)
    missing = set(installation["allowed_tools"]) - set(available)
    if missing:
        raise ValueError(f"Tools no disponibles: {missing}")
    
    # Crear task
    task = await self.db.insert("tasks", {...})
    task_id = task["id"]
    
    try:
        result = await asyncio.wait_for(
            self._execute_agent(installation, input_data),
            timeout=AGENT_EXECUTION_TIMEOUT
        )
        await self._save_outputs(installation_id, task_id, result)
        await self.db.execute("UPDATE tasks SET status = 'completed' WHERE id = $1", task_id)
        return {"task_id": task_id, "status": "completed", "output": result}
        
    except asyncio.TimeoutError:
        await self.db.execute("UPDATE tasks SET status = 'timeout' WHERE id = $1", task_id)
        raise TimeoutError(f"Exceeded {AGENT_EXECUTION_TIMEOUT}s")
```

async def get_agent_output(self, installation_id: str, output_key: str, task_id: str = None) -> dict:
    row = await self.db.fetchrow("""
        SELECT ao.*, at.output_schema
        FROM agent_outputs ao
        JOIN agent_installations ai ON ao.installation_id = ai.id
        JOIN agent_templates at ON ai.template_id = at.id
        WHERE ao.installation_id = $1 AND ao.output_key = $2

BEGIN

## 01 — Modelo de datos

### Arquitectura: Immutable Snapshots

**Problema:** Una actualización sobrescribe el código y rompe instalaciones existentes.

**Solución:** Cada publicación es un snapshot inmutable en `agent_template_versions`.

- Las instalaciones apuntan a `version_id` específico
- Even if developer publishes v2.0, la v1.0 ejecuta el mismo código de su momento
- No hay "update in place" — siempre se crea nuevo snapshot

### 2. Metadata para Pruebas (/test)

**Problema:** No hay campo para input de prueba.

**Solución:** Usar convención dentro de `soul_json`:

```json
{
  "role": "SEO Analyzer",
  "goal": "Analizar URLs...",
  "_metadata": {
    "sample_input": {
      "url": "https://ejemplo.com"
    }
  }
}
```

El endpoint `/test` lee `_metadata.sample_input` y lo pre-carga en el Dashboard.

### 3. JIT Decryption para Credenciales

**Problema:** Backend necesita desencriptar `custom_config` del consumidor.

**Solución:** RPC con SECURITY DEFINER:

```sql
CREATE OR REPLACE FUNCTION get_decrypted_config(installation_id UUID)
RETURNS JSONB
SECURITY DEFINER
SET search_path = admin  -- evitar exposición
AS $$
BEGIN
    -- Desencriptar usando clave de org o Vault
    RETURN decrypt_config((SELECT custom_config FROM agent_installations WHERE id = installation_id));
END;
$$;
```

**Flujo:**
1. Python llama `get_decrypted_config(installation_id)`
2. Solo retorna en memoria efímera del proceso
3. No se guarda en logs ni tablas

### 4. Schema Fingerprinting (Breaking Changes)

**Problema:** auto_update puede romper flujos con breaking changes.

**Solución:** Hash del input_schema antes de actualizar:

```sql
-- Al publicar nueva versión
input_schema_hash = sha256(input_schema::text)

-- Al intentar auto_update
IF new.input_schema_hash != old.input_schema_hash THEN
    -- Bloquear auto_update
    UPDATE agent_installations 
    SET update_policy = 'manual' 
    WHERE id = installation_id;
    -- Notificar al consumidor
END;
```

### 5. Namespacing Jerárquico (Slugs)

**Problema:** Dos orgs wanting the same slug.

**Solución:** Slug es único por owner (no global).

- Display: "SEO Analyzer" (friendly)
- URL interna: `/marketplace/{org_slug}/{agent_slug}`
- Resolución técnica siempre única por `owner_org_id + slug`

### 6. Atomic SQL Increments (Monetización)

**Problema:** Concurrency cause tracking errors.

**Solución:** Trigger de base de datos, no Python:

```sql
CREATE OR REPLACE FUNCTION update_usage_on_task()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.installation_id IS NOT NULL THEN
        -- Atomic increment de uso del mes
        INSERT INTO agent_usage (installation_id, runs, period_month)
        VALUES (NEW.installation_id, 1, to_char(now(), 'YYYY-MM'))
        ON CONFLICT (installation_id, period_month)
        DO UPDATE SET runs = agent_usage.runs + 1;
        
        -- Atomic increment de run_count del template
        UPDATE agent_templates 
        SET run_count = run_count + 1 
        WHERE id = (SELECT template_id FROM agent_installations WHERE id = NEW.installation_id);
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_update_usage
AFTER INSERT ON tasks
FOR EACH ROW EXECUTE FUNCTION update_usage_on_task();
```

**Beneficio:** Tracking 100% preciso incluso bajo carga masiva.

```python
# src/flows/base_flow.py

AGENT_EXECUTION_TIMEOUT = 300       # 5 min hard timeout
AGENT_HEARTBEAT_INTERVAL = 30         # 30s heartbeat
AGENT_HEARTBEAT_DEADLINE = 120       # 2 min sin heartbeat = muerto

async def run_installed_agent(self, installation_id: str, input_data: dict) -> dict:
    task_id = await self._create_task(installation_id, input_data)
    
    # Watchdog que monitorea heartbeat
    watchdog = asyncio.create_task(self._task_watchdog(task_id))
    
    try:
        result = await asyncio.wait_for(
            self._execute_with_heartbeat(task_id, input_data),
            timeout=AGENT_EXECUTION_TIMEOUT
        )
        watchdog.cancel()
        return result
    except asyncio.TimeoutError:
        await self._fail_task(task_id, f"Execution timeout")
        raise

async def _execute_with_heartbeat(self, task_id: str, input_data: dict) -> dict:
    heartbeat_task = asyncio.create_task(self._send_heartbeat(task_id))
    try:
        return await self._execute_agent_logic(input_data)
    finally:
        heartbeat_task.cancel()

async def _send_heartbeat(self, task_id: str):
    while True:
        await self.db.execute("""
            UPDATE tasks SET last_heartbeat = NOW() WHERE id = $1
        """, task_id)
        await asyncio.sleep(AGENT_HEARTBEAT_INTERVAL)

async def _task_watchdog(self, task_id: str):
    while True:
        await asyncio.sleep(AGENT_HEARTBEAT_INTERVAL)
        row = await self.db.fetchrow("""
            SELECT last_heartbeat, status FROM tasks WHERE id = $1
        """, task_id)
        if row["status"] != "running":
            return
        if row["last_heartbeat"]:
            dead_since = datetime.utcnow() - row["last_heartbeat"]
            if dead_since.total_seconds() > AGENT_HEARTBEAT_DEADLINE:
                await self._fail_task(task_id, "Agent heartbeat lost")
                return
```
